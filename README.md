This work was done by Charlz Galdo.

Week 1
We added a linux host and changed it's IP address through the config menu. I learned how to set up an IP address for a computer.
Week 2
We set up a Lan and configured static IP addresses using 3 different methods, Editing the config using the GNS3 menu, using nano to edit /etc/network/interfaces and ip addr add command. The ip addr add command only set a temporary IP. I learned The different ways to set the IP.
Week 3
We used netcat to create a simple client to server connection between two linux hosts and exchanged messages. We also did a network capture and exported the pcap file using filezilla. I learned how data flows through the application level.
Week 4
We built a network with 2 subnets and a router, configured the IPs and gateways, enabled port forwarding and tested the routing using ping for task 1. For task 2 we used the prebuilt OSPF network, viewed routing tables in the network and used traceroute to observer paths. I learned that removing a link on one path made it go through another path to complete sending its data from one host to another. I also learned how to use subnets and how to port forward.
Week 5
We created VLANs on a switch by setting up different VLAN IDs to each port and looked at how the communications were blocked between groups for task 1. In task 2, we then added a router and set up trunk ports and VLAN sub-interfaces to enable communcations between the VLANs. I learned how VLANs operate and how to allow communications within the network.
Week 6
We explored how ARP maps the ip addresses to the MAC addresses by looking through the ARP tables and observing the entry changes after pinging between hosts in task 1. I learned how devices map out addresses dynacmically during communications. In task 2 we built a multi-subnet network and configured them to enable communications between networks.I learned how traffic is routed between two networks from setting up gateways, forwarding and testing the connections with pings.
